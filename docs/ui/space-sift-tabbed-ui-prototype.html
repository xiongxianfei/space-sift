Safety model

Make the safety rules visible instead of hiding them in copy. Users should know why some operations are unavailable.

Unprivileged by default

Normal scanning and history flows stay in the standard desktop process.

Recycle Bin first

Recoverable deletion is the default. Permanent deletion requires an explicit advanced confirmation.

Local-only history

Completed scans and cleanup logs are local to SQLite-backed app storage.

Best practice: disable or explain unavailable resume actions with can_resume, not a separate UI-only capability flag.

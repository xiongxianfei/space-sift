# Execution plan template

Use this format for any approved plan stored in `docs/plans/`.

## Header

- Title
- Status: proposed | active | blocked | done | superseded
- Owner
- Start date
- Last updated
- Related issue or PR
- Supersedes

## Goal

State the concrete user or maintainer outcome.

## Why now

Explain why this work matters and what problem it solves.

## Scope

### In scope
- item

### Out of scope
- item

## Constraints

- technical or product constraint
- rollout or compatibility constraint

## Milestones

1. Milestone name
   - deliverable
   - verification

2. Milestone name
   - deliverable
   - verification

## Progress

Keep this current while implementing. Prefer dated notes.

- YYYY-MM-DD: progress update

## Decision log

Record important decisions and the reason for each one.

- YYYY-MM-DD: decision -> rationale

## Surprises and discoveries

Record anything learned during implementation that future work should know.

- YYYY-MM-DD: note

## Validation notes

Record what was actually verified.

- command: result
- command: result

## Risks and follow-ups

- risk
- deferred follow-up
